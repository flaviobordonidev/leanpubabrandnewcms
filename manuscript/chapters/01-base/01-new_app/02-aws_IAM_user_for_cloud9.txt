# L'ambiente di sviluppo AWS come IAM user

Non è una buona idea lavorara su cloud9 come root. Lo stesso Amazon Web Server consiglia di creare un utente IAM per gestire cloud9.

Tra le varie ragioni due sono da evidenziare:

* mantenere la sicurezza
* l'utente root non può essere scelto come collaboratore per altri ambienti aws cloud9.


ATTENZIONE:
l'utente IAM è visto come una persona differente dall'utente principale "root" e quindi avrà un suo ambiente cloud9 e le app create come "root" non saranno visibili nell'ambiente dello IAM user e viceversa.

E' quindi importante scegliere con cura l'utente:

* se si vogliono raggruppare le app fatte per i vari clienti si possono usare IAM users con il nome dei nostri clienti
* se voglio vedere tutte le app uso un unico utente 


ATTENZIONE:
Con lo IAM user si possono incorrere in errori per mancanza di autorizzazioni. Cosa che con "root" non avviene perché ha accesso a tutto.

## Effettuiamo login come nuovo IAM utente Cloud9

Guardiamo lo User ARN -> arn:aws:iam::93703132901:user/cloud9

Il numero è usato per il login come utente


Effettuiamo il logout come amministratore

Nel login scegliamo -> "Sign in to a different account"
